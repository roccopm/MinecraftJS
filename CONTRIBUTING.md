# Contributing to MinecraftJS

Thank you for your interest in contributing! We welcome help from everyone, regardless of experience level. Every contribution counts, whether it's a bug report, a suggestion, or a code change.

## How You Can Help

- **Report bugs** – Found something broken or unexpected? Please open an issue. The more detail you can provide (steps to reproduce, what you expected, etc.), the better.
- **Suggest ideas** – Have an idea for a feature or improvement? We'd love to hear it. Open an issue and describe your suggestion.
- **Submit code** – Fixes and features are all welcome.

## Development Setup

See the [README](README.md#local-development) for how to run the game locally (Node.js or Docker).

## Pull Request Process

1. Fork the repo and create a branch for your change.
2. Make your changes and test them locally.
3. Open a pull request with a clear description of what you changed and why.
4. We'll review and may ask for small tweaks. Once everything looks good, we'll merge.

## Code and Tools

- **AI assistance** – Using AI tools to help you code, debug, or learn the codebase is fine. We do ask that you understand what you're submitting and that your contributions are intentional and coherent with the project - not just bulk-generated or unreviewed output.
- **Style** – Ensure that your code matches the formatting standards displayed in the repo. Follow best practices and write readable code!
